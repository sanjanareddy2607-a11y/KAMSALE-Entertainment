# Kamsale Entertainment

Official website for Kamsale Entertainment — a Karnataka-based cultural events company offering authentic Kamsale folk performances for weddings, temple functions, corporate events, and festivals. The site also runs **Vinootana Golden Singers**, a statewide online audition and registration drive for specially-abled performers, built directly into the same project.

## Tech Stack

### Frontend
- **React 19** with **TypeScript** — component-based UI
- **Vite** — dev server and build tool
- **Tailwind CSS v4** — utility-first styling
- **Framer Motion** — page transitions, swipeable carousels, and micro-animations
- **React Router v7** — client-side routing (home, registration page, success page)
- **Lucide React** — icon set
- Custom bilingual content system (`LanguageContext.tsx`) — toggles all site text between **English** and **Kannada** without any third-party i18n library

### Backend
- **Node.js** with **Express 5** — REST API server
- **TypeScript**, run directly via **tsx** (no separate compile step in dev)
- **Multer** — handles multipart file uploads from the registration form
- **CORS** + **dotenv** for configuration

### Database
- **SQLite**, accessed via **better-sqlite3** — a single local file (`data/registrations.db`) storing all registration submissions
- Includes a duplicate-registration guard: a unique index plus an application-level check blocks the same guardian phone number or email from registering twice for the same event

### Email
- **Resend API** — transactional email notifications sent on every new registration
- Chosen specifically because the app is hosted on **Render's free tier**, which blocks all outbound SMTP ports (25/465/587). Resend sends over HTTPS, which Render never blocks.

### File Storage (for large media)
- Participant photos and audition recordings are collected through an **external Google Form** (linked from the registration flow) rather than uploaded directly to the server, to avoid file size limits on the free hosting tier. Files land in Google Drive; a **Registration ID** generated by the database links each Drive submission back to its database record.

### Hosting / Deployment
- **Render** (free tier) — hosts both the Express API and the built React frontend from a single service
- Custom domain support via **GoDaddy** DNS, pointed at the Render service

## Project Structure

```
│   ├── context/

└── uploads/                 # Server-side temp file storage (gitignored)
```

## Getting Started

Install dependencies:

```bash
npm install
```

Run the frontend and backend together in development:

```bash
npm run dev:all
```

Or run them separately:

```bash
npm run dev          # Vite frontend only — http://localhost:5173
npm run dev:server   # Express backend only — http://localhost:3001
```

## Environment Variables

Create a `.env` file in the project root:

```env
# Resend API (email notifications)
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxx

# Where registration notification emails are sent
RECIPIENT_EMAIL=your-email@example.com

# Server port
PORT=3001
```

Get a free Resend API key at [resend.com](https://resend.com). On the free tier, verify your recipient email under **Emails → Add Email Address** before testing.

## Build & Production

```bash
npm run build      # type-checks and builds the frontend into dist/
npm run start:server  # runs the Express server (serves dist/ + API)
```

On Render, set the start command to `npm run start:server` and add the environment variables above under the service's **Environment** tab.

## Database Notes

Registrations are stored in a single SQLite file. This works well locally but **does not persist on Render's free tier**, since the disk resets on every redeploy or sleep/wake cycle. For production data that must survive restarts, migrate to a hosted database (e.g. AWS RDS PostgreSQL, Render's managed Postgres, or Supabase) and update `server/db.ts` accordingly.

## Bilingual Content

All translatable site text lives in `src/context/LanguageContext.tsx`, split into matching `EN` and `KN` objects with identical structure. Components read text via the `useContent()` hook rather than importing strings directly, so any component using `useContent()` automatically updates when the user toggles language.
